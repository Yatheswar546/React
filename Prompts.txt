18th Concept: useRef

================================================================================================================
notes.txt 
        "NO UNNECESSARILY RE-RENDERS"
=> If there are more re-renders in a short-span of time then the browser will run slow or get crashed.
=> A setter method always re-render a component, below is a example.

Eg: App.js
import { useState } from "react";

export default function App(){
    // 1st approach: Use simple Js variable
    let counter = 0;
    // 2nd approach: Use useState variable
    const [count, setCount] = useState(0);

    const handleIncrement = () => {
        setCount((prevValue) => prevValue + 1);
        counter++;
        console.log("count: ", counter);
    };

    console.log("Component re-rendered");

    return(
        <div className="App">
            <h2>{count}</h2>
            <button onClick={handleIncrement}>Increment</button>
        </div>
    );
}

=> To solve the above re-rendering problem the below approaches won't work effectively.
    1st approach: Use simple Js variable
    2nd approach: Use useState variable
=> To solve this issue we use "useRef" hook. (3rd approach)

useRef:
=> "useRef" is used to create mutable reference persisting across multiple re-render for an element while avoiding unnecessary re-renders.
=> The useRef will updates the component(data variable) without re-rendering.
=> The value will be persistent across multiple re-renders which was not possible with 'useState'.

Demo code: App.js
import { useState } from "react";

export default function App(){

    // 3rd approach: useRef
    const counterRef = useRef(0);

    const handleIncrement = () => {
        counter++;
        console.log("count: ", counter);

        // writing a useRef variable
        counterRef.current++;
        console.log("useRef variable value", counterRef.current);  // reading a useRef variable
    };

    console.log ("Component re-rendered");

    return(
        <div className="App">
            <h2>{count}</h2>
            <button onClick={handleIncrement}>Increment</button>
        </div>
    );
}

NOTE: the reading and writing of a useRef variable can be done only at two places.
        1. handler functions 
        2. useEffect hooks 

Pitfalls of useRef Hook:
=> Do not read/write ref.current during rendering.
=> React expects that components should behave like a pure function.
=> If the inputs(props, state and context) are the same, it should return exactly the same JSX.

Use cases of useRef Hook:
=> DOM Manipulation
=> Focusing a text input

Eg: 
import { useRef } from "react";

export default function Form(){
    const inputRef = useRef(null);

    function handleClick(){
        inputRef.current.focus();
    }

    return(
        <>
            <input ref={inputRef} />
            <button onClick={handleClick}>
                Focus on input
            </button>
        </>
    );
}


When to use "useState" and "useRef".
=> If we want to use a variable in jsx i.e. that value is shown on UI then we should use "useState".
=> For the variables which we won't use for UI but use for sub-calculations and other operations we need "useRef".
