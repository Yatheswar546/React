useMemo:

=> Expensive function: A function that is executed multiple times but gives same input increasing bandwidth.
Eg: 
function square(num){
    console.log('Square ', num);
    return num * num;
}

=> To avoid expensive function we use "caching function" that helps us avoid operations that gives same values for same inputs.

Eg:
function memoizeFunction(fn){
    const cache = {};

    return function (input){
        if(cache[input]){
            console.log('Fetching from cache...');
            return cache[input];
        } else {
            const result = fn(input);
            cache[input] = result;
            return result;
        }
    };
}

const memoizedFn = memoizeFunction(square);

In the above code, memoizeFunction function stores the computed value, 
and if the input is already calculated, it gives the result instead of re-calculating it.

Disadvantages:
=> You have to declare a new variable to store input and results.
=> You have to check if the value is already calculated.
=> You have to assign values and store once they’re calculated.

Solution:
useMemo can help us achieve the same "memoization" with less lines of code in an optimized manner.

3 types of Memoization:
1. Memoization of values.
2. Memoization of function handlers.
3. Memoization of entire component.

Uses of "useMemo":
=> By using useMemo, React automatically handles the caching for us.
=> It ensures that computation is only performed when the input prop changes.
=> Avoiding unnecessary computations on subsequent renders with the same input value.
=> useMemo actually returns calculated values and memoize that value.

How to memoize functions or Child components ?

SquareComponent.jsx

import React from "react";

function SquareComponent({ handler, input }) {
    const square = (num) => {
        console.log("Square ", num);
        return num*num;
    }

    return (
        <>
            <h1>{square(input)}</h1>
            <button onClick={handler}>Increment</button>
        </>
    );
}

export default (SquareComponent);

In the above SquareComponent, it receives two props:
1) input: Number to calculate square
2) handler: function to update the input value

App.jsx:

import { useState, useCallback, useRef, useMemo } from "react";

export default function ParentComponent() {
    const [state, setState] = useState(false);
    const [input, setInput] = useState(1);
    console.log("Parent Component Rendered");

    const handler = (prevInput) => setInput(prevInput + 1);

    const statehandler = () => {
        setState(!state);
    };

    return (
        <>
            <button onClick={statehandler}>
                Change State of Parent Component
            </button>
            <SquareComponent handler={handler} input={input} />
        </>
    );
}

In the above ParentComponent, it has two state variables:
1) state : To re-render component
2) input: Number to calculate square

In tha above example, when we click on "Change State of Parent Component" button.
Along with Parent Component, Child component is also re-rendered which is an "unnecessary re-render"

We can optimize this behavior using "memoize" technique

===============================================

Step 1: Memoize the child component (Square component)

=> Components only re-renders if their "state" or "props" changes
=> So, we can memoize the child component using "React.memo"

React.memo:

=> React.memo is a higher order component (HOC) provided by React that
    performs a shallow equality check on the component's props.
=> It determines if the component should be re-rendered by comparing 
    the previous and current props.
=> If the props haven't changed, React.memo prevents the re-rendering of the component.

Example:

function SquareComponent({ handler, input }) {
    const square = (num) => {
        console.log("Square ", num);
        return num;
    }

    return (
        <>
            <h1>Square Component</h1>
            <button onClick={handler}>Increment</button>
        </>
    );
}

export default React.memo(SquareComponent);

In the above code, we use "React.memo" but still we are facing the same issue.
Because in SquareComponent when we pass the props(handler) as handler is a function the function reference is passed as Pass by Reference concept.
Whenever we call ParentComponent, a new memory location is created for handler function and the new memory reference is passed to child component.

===============================================

To fix the above issue we can use "useCallback" hook.

useCallback:

=> useCallback is a React Hook that lets you cache a function definition between re-renders.
=> useCallback function is used to memoize functions and avoid calling them unless certain dependencies changes

Example:
const handler = UseCallback(() => {
    setInput(input + 1);   
}, [input]);


NOTE:
useMemo      --> memoize the value 
useCallback  --> memoize the function object, function, arrays references

Prop Drilling:

=> Components used to get props that they don't need it

Let us consider a scenario, we have "n" components
component A -> has state A 
component B (Child of component A)
component C (Child of component B)
component D (Child of component C)
component E (Child of component D)
.
.
.
.
.
component n (Child of previous component) -> it requires state A 

Now, in the above scenario "component n" requires state A from "component A". 
It has to pass through all components B, C, D, E..... upto "n" even though the middle components don't need state A.

The above scenario is called "Prop Drilling".

Example: on Prop-Drilling

App.jsx
import React, { useState } from "react";
import ParentComponent from './ParentComponent';

function App() {
    const [count, setCount] = useState(0);

    return (
        <>
            <ParentComponent count={count} setCount={setCount} />
        </>
    );
}

We can fix this using "useContext" hook.

===============================================

useContext:

=> useContext will automatically pass all the state and data variables declared in the context to all the child components of that ParentComponent.

=> "createContext" lets you create a 'context' that components can provide or read.

=> we will create a context which holds a value at global and that value can be used at any component 
=> Provider :
=> Consumer :

Steps to use "useContext" Hook:

1. Create a Context with "useContext" hook in React.

CountContext.jsx 
import React from 'react';

export const CountContext = React.createContext();

2. Wrap App.js with "contextProvider" by passing values that need access in all the children components.

App.jsx
import React, { useState } from 'react';
import { CountContext } from './CountContext';
import ParentComponent from './ParentComponent';

function App() {
    const [count, setCount] = useState(0);

    return (
        <div>
            <CountContext.Provider value={{ count, setCount }}>
                <ParentComponent />
            </CountContext.Provider>
        </div>
    );
}

3. Now we can directly access the count variable declared in App.js component in any component (GrandchildComponent.js)
    without passing it as props through every children component

import React from 'react';

        </div>

useLocalStorageInput.jsx (inside hooks folder)

import { useState, useEffect } from 'react';

export const useLocalStorageInput = (key, defaultValue) => {

    const [input, setInput] = useState(localStorage.getItem(key) ?? defaultValue);

    useEffect(() => {
        localStorage.setItem(key, input);
    }, [input]);

    return {input, setInput} ;

};

Example 2 (Custom Hook that returns "window size"):

import { useState, useEffect } from 'react';

const useWindowSize = () => {
    const [windowsize, setWindowSize] = useState({
        width: window.innerWidth,
        height: window.innerHeight
    });

    useEffect(() => {
        const handleResize = () => {
            setWindowSize({
                width: window.innerWidth,
                height: window.innerHeight
            });
        };

        window.addEventListener("resize", handleResize);

        return () => {
            window.removeEventListener("resize", handleResize);
        };
    }, []);;

    return windowsize;
};

export default useWindowSize;

Example 3 (useLocalStorage custom Hook):

import { useState, useEffect } from 'react';

const useLocalStorage = (key, initialValue = "") => {

    const [value, setValue] = useState(() => {
        const storedValue = localStorage.getItem(key);
        
        return storedValue ? JSON.parse(storedValue) : initialValue;
    });

    useEffect(() => {
        localStorage.setItem(key, JSON.stringify(value));
    }, [key, value]);

    return [value, setValue];
};

export default useLocalStorage;
