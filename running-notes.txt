Problem Statement: infinite loop problem

import React, { useState } from "react";

export default function Products(){
    const [products, setProducts] = useState([]);

    function fetchData(){
        fetch("https://dummyjson.com/products")

useEffect: used to handle side-effects in React application.
=> In the above example "fetchAPI" which is outside of React is making the website slow by infinite loop.
=> setTimeouts, LocalStorages, Read/Write files etc...

Syntax: useEffect((callback) => { Dependency Array})

Eg: without Dependency list
    
    useEffect(() => {
        console.log(products, "products");
        fetchData();
    });

Eg: with Dependency list

    useEffect(() => {
        console.log(products, "products");
        fetchData();
    }, []); //mounting

    In this fetchData() renders only once, because of Dependency List 
    This is also known as "Mounting Phase".

Solution: Promise Concurrency
            |-> Promise.all()
            |-> Promise.allSettled()
            |-> Promise.any()
            |-> Promise.race()

let consider timetaken for 3 tasks 
1 ==> 3 secs
2 ==> 5 secs
3 ==> 4 secs 

In normal execution : 3 + 5 + 4 = 12 secs

In Promise Concurrency:

after 3 secs 
1 ==> done 
2 ==> 2 secs 
3 ==> 1 sec 

after 4 secs 
1 ==> done 
2 ==> 2 secs 
3 ==> done

after 5 secs 
1 ==> done 
2 ==> done 
3 ==> done

let postApi = await fetch("https://........./posts");
let userApi = await fetch("https://........./users");
let commentsApi = await fetch("https://........./comments");
    
async function fetchData(){
    let startTime = new Date();
    
    let [postRes, userRes, commentRes] = await Promise.all([
        fetch(postApi),
        fetch(userApi),
        fetch(commentsApi),
    ]);

    let [posts, users, comments] = await Promise.all([
        postRes.json(), 
        userRes.json(), 
        commentRes.json(),
    ]);

    console.log(posts, users, comments, "res");

    let endTime = new Date();
    let timeTaken = endTime - startTime;
    console.log(timeTaken);
}

export { fetchData };

NOTE: Promise.all() accepts an array and returns an array.

Debouncing:
=> It is a programming practice that ensures a func. is not called too frequently 
    by delaying its execution until a specified time has passed since the last invocation.
=> Use Cases: Search Autocomplete, Window Resizing, Button clicks etc...

Problem Statement: 
In general in search input fields whenever we type a single letter the API call is done multiple times.
Because of this API is called multiple times increasing in bandwidth and decreasing the performance.

Solution: (Debouncing)
=> Debouncing is used to make a particular function to run after a specific time reducing the load.

Task: 
Create a search bar in a header without debouncing.
Observe the API call made everytime we type.
Later apply Debouncing and observe the change.
(Use axios get method to fetch the details of posts/videos when typing in search bar
  and useEffect() method)

Example code: With Debouncing

function Header(){
    const [searchKey, setSearchKey] = useState("");
    const [timer, setTimer] = useState("");

    const handleChange = (e) => {
        setSearchKey(e.target.value);
    };

    const fetchVideos = async () => {
        try{
            let res = await axios.get(
                `https://................videos?title=${searchKey}`,
            );
            console.log(res.data);
        } catch (err) {
            console.log(err, "error");
        }
    };

    function debounceSearch(func, delay){
        if(timer) clearTimeout(timer);
        let timerId = setTimeout(() => {
            fetchVideos()
        }, delay);
        setTimer(timerId);
    }

    useEffect(() => {
        debounceSearch(1000);
    }, [searchKey]);   // calls the function only when searchKey is updated. known as "update phase in useEffect".

    return(
        <TextField
            type="text"
            placeholder="Search here..."
            value={searchKey}
            onChange={(e) => handleChange(e)}
        />
    )
}

Lifting State Up:

Problem Statement:
The videos we are searching in search bar in Header.js should be updated in videos section in Videos.js
But videos and header are not parent-child so props won't work in this case.
So, how to achieve this ?

Solution:
Header and Videos are siblings to each other in App.js
We can share the data to common parent (App.js) and then share the data to children (Header & Videos).
This is called as "Lifting the State Up".

Example Code:
App.js:

function App(){

    const [searchKey, setSearchKey] = useState("");
    const [timer, setTimer] = useState("");
    const [videos, setVideos] = useState([]);

    const url = searchKey
        ? `https://................videos?title=${searchKey}` : 
        : `https://................videos`;

    const fetchVideos = async () => {
        try{
            let res = await axios.get(url);
            console.log(res.data);
            setVideos(res.data.videos);
        } catch (err) {
            console.log(err, "error");
        }
    };

    function debounceSearch(func, delay){
        if(timer) clearTimeout(timer);
        let timerId = setTimeout(() => {
            fetchVideos()
        }, delay);
        setTimer(timerId);
    }

    useEffect(() => {
        debounceSearch(1000);
    }, [searchKey]); 

    return(
        <>
            <Header searchKey={searchKey} setSearchKey={setSearchKey} />
            <Videos videos={videos}/>
        </>
    )

}

"bind" method:
=> When we pass the method as callback, the value of this is lost and when the executor function executes the callback, 
    this is not pointing to the student object anymore.

    Eg: const student = {
            fullName: "Rohit",
            greet: function(){
                console.log("Hello, " + this.fullName);
            },
        };

        function executeObjectFunction(callback){
            callback();
        }

        executeObjectFunction(student.greet);

        // The above code's output is : Hello, undefined.


=> what we can do is pass the value of "this" explicity to the "student.greet" method.
=> To achieve this we can use "bind" method which can be applied on all methods.

    Eg: const greet = student.greet.bind(student); // bind the object to thee method
        executeObjectFunction(greet); // Passing the object method as callback

=> The bind method returns a new function.
=> It helps us set the value of this to the passed value for specific funtion.

    Eg: const student = {
            fullName: "Rohit",
            greet: function(){
                console.log("Hello, " + this.fullName);
            },
        };

        function executeObjectFunction(callback){
            callback();
        }

        const greet = student.greet.bind(student)
        executeObjectFunction(greet);
