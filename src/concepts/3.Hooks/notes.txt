Rendering: 

=> Whenever we change the values we need to re-render then only we can 
    see the changes.

Eg: Let us create a Counter App which has a variable "count". 
    The variable count stores the value but it won't display on screen 
    as it is a local variable.

Example Code:

export default function Counter() {

    let count = 1;
    
    function handleCounter(){
        console.log(count++);
    }
    
    return (
        <div>
            <h1>Counter App</h1>
            <button onClick={handleCounter}>+</button>
            <h2>{count}</h2>
        </div>
    )

}

=> In the above example the count will increase on console but won't change on screen.
    As it is a local variable, we can't rely on local variables as they can't re-render.

Stateful and Stateless Components:

State:  Holding a state (Eg: counter value, dropdown-open, dropdown-close)
        It will store the data and show the changes on screen.

Stateful : Components with state used when to add a functionality (Eg: to increase the count value)

Stateless : Components without states just show on screen.

=> Class components has default "State" whereas 
    Functional components doesn't.

============================================================================================

Hooks:

=> To resolve the above problem with functional components 
    "hooks" got introduced.

=> Hooks are pre-written functions used in case of State.

=> Hooks has state and which are used for functional components.

Definition: 
Hooks are the functions that allow you to "hook into" React state and lifecycle features from functional components. 
Before Hooks (introduced in React 16.8), these features were only available in class components.

"Big 4" Hooks

1) useState   : "Adds state" to your component (eg: counter value, form input)

2) useEffect  : "Handles side effects" like API calls, timers, or manual DOM updates.

3) useContext : "Consumes global data" (themes/user info) without passing props manually through every level.

4) useRef     : "Creates a reference" to a DOM element or a value that persists without triggering a re-render.

============================================================================================

useState():

=> It stores the a state value to be managed within a function.
=> It returns an array of elements [latest-state-value, function to be update].
=> We need to use function to update state values.

    useState[state, setState]

=> In functional components we can have many "states" with different names.
=> we can't update the counter directly we need to update the function to change the state-value.
=> setState() re-renders the component after updating the state.

Syntax:  let[state, setState]  =  useState(default-value)

Example Code:

// In Counter.jsx

import { useState } from "react";

export default function Counter() {

    let [counter, setCounter] = useState(0);

    // 1st Method of Handling State
    function handleCounter() {
        setCounter(counter + 1);
    }

    return (
        <div>
            <h1>Counter App</h1>

            <button onClick={handleCounter}>+</button>

            {/* 2nd Method of handling state */}
            <button onClick={() => { setCounter(counter - 1) }}>-</button>

            <h2 className="counter">{counter}</h2>


        </div>
    )

}
